# AI 모델 정책 관리

## 1. 개요
- **화면명**: AI 모델 정책 관리
- **목적**: 세차 오더 생성 및 자동 검수 로직에 사용되는 AI 모델의 파라미터(임계치 등)와 활성화 상태를 관리합니다.
- **주요 기능**: 모델별 정책 목록 조회, 정책 상세 정보 확인 및 수정(Drawer 기반), 상태별 필터링.

## 2. 기능 명세

### 2.1 검색 및 필터
목록 상단의 필터를 통해 관리 대상 모델을 선별할 수 있습니다.

| 구분 | 필드명 | 타입 | 설명 및 로직 |
| :--- | :--- | :--- | :--- |
| **검색** | 통합 검색 | Input | **대상 필드 (OR 조건)**:<br>- 모델명<br>- 모델 ID |
| **필터** | 모델 유형 | Select | 분류(Classification), 탐지(Detection), 세그멘테이션 등 |
| **필터** | 활성 상태 | Select | 전체, 활성, 비활성 |

### 2.2 AI 정책 목록
필터링된 AI 모델 정책 데이터를 테이블 형태로 표시합니다.

- **데이터 소스**: 확인 가능
- **행 식별자**: 확인 가능

#### 컬럼 정의
| 헤더명 | 비고 |
| :--- | :--- |
| **모델 ID** | AI 모델의 고유 식별자 |
| **모델명** | 모델의 명칭 |
| **모델 유형** | 적용된 AI 기술 구분 |
| **버전** | 현재 적용 중인 모델 버전 |
| **임계치(Threshold)** | 자동 판정 기준이 되는 Confidence Score (0.0 ~ 1.0) |
| **상태** | 활성/비활성 상태별 배지 색상 적용 |
| **업데이트 일시** | 마지막 정책 수정 시각 (YYYY-MM-DD HH:mm) |

### 2.3 정책 상세 및 수정
목록의 행을 클릭하면 우측 Drawer에서 상세 정보를 제공하고 정책을 수정할 수 있습니다.

- **진입 트리거**: 테이블 행(Row) 클릭
- **제공 정보**:
  - **기본 정보**: 모델 ID, 명칭, 버전 정보, 모델 설명.
  - **파라미터 설정**:
    - **Confidence Threshold**: 판정 기준 점수 (Input).
    - **가중치 설정**: 특정 환경(야간, 우천 등)에 대한 가중치 적용 여부.
  - **적용 범위**: 전체 적용 또는 특정 지역/존(Zone) 단위 적용 설정.

#### 처리 동작
- **저장**: 수정된 파라미터 값을 저장하고 즉시 운영 로직에 반영합니다.
- **상태 변경**: 모델의 활성/비활성 상태를 토글할 수 있습니다.
- **감사 로그**: 수정 시 변경 사유 입력을 필수로 하며, 관리자 ID와 함께 기록됩니다.

## 3. 특이사항
- **일괄 수정**: 여러 모델을 선택하여 활성 상태를 한꺼번에 변경하는 기능을 지원할 수 있습니다.
- **권한 관리**: 정책 수정 권한은 '운영 관리자' 이상의 등급에서만 허용됩니다.
