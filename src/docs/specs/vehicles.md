# 차량 관리

## 1. 개요
- **화면명**: 차량 관리
- **목적**
  - **수행률 모니터링**: 오더가 발행되더라도 수행이 100% 보장되지 않으므로, 수행률이 낮은 차량 및 존별 현황을 집중 모니터링합니다.
  - **운영 계획 수립**: 존별 차량 보유 대수를 파트너사에 제공하여, 적정 수행원 배정 규모를 계획할 수 있도록 지원합니다.
- **주요 기능**: 차량 목록 조회, 다차원 필터링, 차량 상세 정보 및 세차 이력 조회.

## 2. 기능 명세

### 2.1 검색 및 필터
상단 카드 영역에 위치하며, 리스트 데이터를 실시간으로 필터링합니다.

| 구분 | 필드명 | 타입 | 설명 및 로직 |
| :--- | :--- | :--- | :--- |
| **검색** | 통합 검색 | 입력창 | **대상 필드**: 차량번호, 존이름, 존 ID (대소문자 구분 없음) |
| **필터** | 지역1 | 선택창 | 지역1 기준 필터링. 변경 시 지역2 필터 초기화. |
| **필터** | 지역2 | 선택창 | 선택된 지역1에 속한 지역2 목록만 노출. |
| **필터** | 파트너명 | 선택창 | 파트너명 기준 필터링. |
| **초기화** | 필터 초기화 | 버튼 | 모든 검색어 및 필터 조건을 초기화합니다. |

### 2.2 차량 목록
필터링된 결과를 테이블 형태로 표시합니다.

- **데이터 소스**: 확인 가능
- **행 식별자**: 확인 가능
- **정렬**: 원본 데이터 순서 유지

| :--- | :--- |
| **차량번호** | |
| **존이름** | |
| **존 ID** | |
| **지역1** | |
| **지역2** | |
| **파트너명** | |
| **진행중 오더** | 차량에 활성화된 오더가 있는 경우, 해당 오더의 **진행상태**를 노출 (클릭 시 오더 상세 이동) |
| **마지막 세차일** | YYYY-MM-DD 형식 |

### 2.3 차량 상세
목록의 행을 클릭하면 우측에서 슬라이딩되는 상세 창입니다.

- **진입 트리거**: 테이블 행 클릭
- **타이틀**: 차량 상세 - {차량번호}

#### 섹션 구성
1. **기본 정보**
   - **차량번호**
   - **차종**
   - **연료유형**
   - **존이름**: {존이름} ({존 ID})
   - **지역1**: {지역1}
   - **지역2**: {지역2}
   - **파트너명**
   - **진행중 오더**: 해당 오더의 **진행상태** 노출 (없을 시 "없음")
   - **마지막 세차일**
   - **세차 경과일**: {세차경과일}일

2. **세차 이력**
   - 현재 프로토타입 단계에서는 예시 데이터 리스트를 표시함.
   - 각 이력 클릭 시 해당 오더 상세 화면을 새 창으로 연결함.

## 3. 제품 정책 및 데이터 로직
- **세차 대상 기준**: 차량의 sharing_type을 기준으로 하며, 현재 **[단기(socar), 시승(testdrive)]** 타입의 차량에 대해서만 세차를 수행합니다.
- **수행 현황 관리**: 오더 발행이 곧 수행 완료를 의미하지 않으므로, '진행중 오더'의 상태값을 통해 실제 수행 여부를 추적합니다.
- **경과일 계산**: 오늘 날짜에서 마지막 세차일을 뺀 일수를 계산하여 표시합니다.
