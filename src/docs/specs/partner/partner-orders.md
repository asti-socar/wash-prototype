# 오더 조회

## 1. 개요
- **화면명**: 오더 조회
- **목적**: 자사에 배정된 세차 오더를 읽기 전용으로 조회 및 상세 확인
- **주요 기능**: 조회/필터링, 상세 확인 (Drawer)
- **권한**: 읽기 전용 — 오더 발행/완료/취소 등 상태 변경 불가

### 인터널 오더 관리 대비 차이점
| 항목 | 인터널 (오더 관리) | 파트너 (오더 조회) |
| :--- | :--- | :--- |
| 화면명 | "오더 관리" | "오더 조회" |
| 데이터 범위 | 전체 오더 | 자사 배정 오더만 (`partner === currentPartner.partnerName`) |
| 오더 발행 | 수기 발행 Drawer 제공 | **제거** |
| 수행 완료 | 상세 Drawer에서 완료 처리 | **제거** |
| 오더 취소 | 상세 Drawer에서 취소 (사유 입력) | **제거** |
| 파트너 이름 컬럼 | 있음 | **제거** (단일 파트너) |
| 파트너 유형 컬럼 | 있음 | **제거** |
| 파트너 이름 필터 | 상세 필터 | **제거** |
| 파트너 유형 필터 | 상세 필터 | **제거** |
| 상세 파트너 유형 필드 | 기본 정보 카드에 표시 | **제거** |
| 리스트 컬럼 수 | 16개 | 14개 |
| 상세 필터 항목 수 | 10개 | 8개 |
| Drawer footer | 닫기 + 수행 완료 + 오더 취소 | 닫기만 |
| 대시보드 연동 | quickFilter (인터널 대시보드) | initialFilter (파트너 대시보드) |

## 2. 데이터 정책

인터널 오더 관리의 데이터 정책(섹션 2.1~2.8)을 동일하게 참조합니다. 다음 사항은 파트너 어드민에서 **읽기 전용**으로 표시됩니다.

- 상태 전이 정책 (발행 → 예약 → 수행 중 → 완료, 취소)
- 취소 유형 및 책임 분류 (9종)
- 리드타임 및 지연 처리 (72h 기준)
- 파트너 유형별 정책 (현장/입고/핸들러)
- 점검 유형 매핑 (A/B/C/D)

## 3. 화면 구성 및 기능

### 3.1 헤더
| 요소 | 설명 |
| :--- | :--- |
| 타이틀 | "오더 조회" |
| 서브타이틀 | "{partnerName}에 배정된 오더를 조회합니다." |
| **[목록 다운로드]** | 현재 필터 결과를 Excel 다운로드 |

- 인터널의 **[오더 발행]** 버튼은 제거

### 3.2 리스트

#### 검색 및 필터

**기본 필터** (항상 1줄 노출)

| 필드 | 타입 | 설명 |
| :--- | :--- | :--- |
| 차량 번호 | 텍스트 검색 | 검색 아이콘 포함. 차량 번호 기준 검색 |
| 세차 유형 | 셀렉트 | 내외부, 내부, 외부, 특수, 협의, 라이트, 기계세차 |
| 진행 상태 | 셀렉트 | 발행, 예약, 수행 중, 완료, 취소 |
| 발행 일시 | 날짜 범위 | 시작일~종료일. 기본값: 최근 1개월 |

**상세 필터** ("+ 필터 추가" 버튼으로 온디맨드 추가, 8개)

| 필드 | 타입 | 설명 |
| :--- | :--- | :--- |
| 차종 | SearchableSelect | 차량 모델명 |
| 존 이름 | SearchableSelect | 배정 존 |
| 오더 구분 | 셀렉트 | 긴급, 정규, 변경, 수시, 특별 |
| 발행 유형 | 셀렉트 | 위생장애, 고객 피드백(ML)_긴급, 초장기 미세차, 주기세차 등 |
| 지역1 | 셀렉트 | 시/도 |
| 지역2 | 셀렉트 | 구/군 (지역1 선택 전 비활성화. 지역1 변경 시 초기화) |
| 취소 유형 | 셀렉트 | 진행 상태 '취소' 선택 시 자동 추가. 취소 외 비활성화 |
| 지연 여부 | 셀렉트 | 진행 상태 '완료' 선택 시 자동 추가. 완료 외 비활성화. 지연/정상 |

- 인터널 대비 **파트너 이름**, **파트너 유형** 필터 제거

**필터 UI 동작**

- **필터 추가 플로우**: "+ 필터 추가" 클릭 → 드롭다운에서 항목 선택 → 상세 필터 영역에 추가 & 자동 펼침. 이미 추가된 필터는 드롭다운에서 제외
- **상세 필터 토글**: 1개 이상 추가 시 "상세 필터(N)" 토글 표시. 접기/펼치기 가능
- **자동 추가**: 진행 상태 '취소' 선택 시 취소 유형 자동 추가, '완료' 선택 시 지연 여부 자동 추가
- **개별 제거**: 상세 필터 항목에 마우스 호버 시 X 버튼으로 제거 가능
- **적용 필터 칩**: 값이 설정된 필터를 칩 형태로 표시. 각 칩 X 버튼으로 개별 해제. 2개 이상일 때 "전체 해제" 링크 표시
- **대시보드 필터 칩**: 대시보드에서 연동된 필터는 "대시보드 필터" 라벨로 칩 표시
- **초기화** (↻): 모든 필터 초기값 리셋 + 상세 필터 모두 제거 + 기간 최근 1개월 복원

#### 컬럼 (14개)
오더 ID, 오더 구분, 발행 유형, 세차 유형, 차량 ID, 차량 번호, 차종, 지역1, 지역2, 존 이름, 진행 상태(배지), 취소 유형(취소 시만), 지연 여부(완료 시만), 발행 일시

- 인터널 대비 **파트너 이름**, **파트너 유형** 컬럼 제거
- **상태 배지**: 완료(초록), 발행/예약(주황), 수행 중(파랑), 취소(회색)
- **지연 여부 배지**: 완료 상태일 때만 표시. 지연(빨강), 정상(초록). 그 외 상태는 `-` 표시
- **데이터 건수 표시**: 필터 패널 아래, 리스트 위에 표시. 필터 적용 시 `필터된 결과 N건 / 전체 **M**건`, 필터 미적용 시 `전체 **M**건`
- **기본 정렬**: 발행 일시 내림차순
- **페이지네이션**: 40건

#### 대시보드 연동 (initialFilter)
파트너 대시보드에서 `initialFilter` props로 필터값 전달 (status, orderType, cancelType, delayed)

### 3.3 상세 (Drawer) — 읽기 전용

3개의 탭으로 구성됩니다. 핸들러 파트너는 "미션 및 분실물" 탭 숨김.

- Drawer footer: **닫기** 버튼만 표시 (인터널의 수행 완료/오더 취소 버튼 제거)

#### ① 상세정보 탭
- **기본 정보**: 오더 ID, 차량 ID, 차량 번호, 차종, 존, 지역, 발행/세차 유형, 수행원, 진행 상태, 취소 유형(취소 시만), 발행/수행 일시, 메모
  - 인터널 대비 **파트너 유형** 필드 제거
- **오더 체인 & 리드타임** (완료 오더만): 별도 카드
  - 참조 오더가 있는 경우: 세로 타임라인으로 취소→재발행 체인 시각화
    - 각 노드: 참조 오더 ID, 취소 유형(배지), 귀책/면책 표시, 발행·취소 시각 및 경과시간
    - 마지막 노드: 현재 오더(완료 배지), 재발행·완료 시각
  - 참조 오더가 없는 경우: "직접 완료 (참조 오더 없음)" 표시
  - 산출 요약 박스: 리드타임, 면책 시간, 귀책 소요시간, 판정(지연/정상). 지연 시 빨간 배경
- **탁송 정보** (입고 전용): 입고/출고 예약 ID + 탁송 상태. 별도 카드.
- **미션핸들 예약 정보** (핸들러 전용): 미션핸들 예약 ID + 예약 상태. 별도 카드.
- **금액 및 연계 정보** (핸들러 제외): 최종 청구 금액, 연계 오더(입고 전용)

#### ② 사진 및 점검 탭
- **진행 이력**: 상태 변경 타임라인
- **사진**: 세차 전/후 사진, 기타 사진 (D유형은 세차 전 사진 미노출)
  - 핸들러: 세차 전 1~5장, 세차 후 7~10장, 세차 후기 1~100자
- **점검 리스트**: 유형별(A~D) 점검 결과
  - 이상 항목: 빨간색 '이상' 배지 + 증빙 사진
  - 타이어 관련: 2×2 Grid (FL/FR/RL/RR)

#### ③ 미션 및 분실물 탭
- **미션**: 연동된 미션 목록 및 상태 (읽기 전용)
- **분실물**: 분실물 ID, 접수일시, 구분, 처리상태, 상세 정보 (읽기 전용)

## 4. 운영 시나리오

**오더 현황 확인**: 파트너 대시보드 → 상태 카드 클릭 → 오더 조회 페이지 (필터 자동 적용)

**오더 상세 확인**: 리스트에서 행 클릭 → Drawer 오픈 → ① 기본 정보 확인 → ② 사진/점검 확인 → ③ 미션/분실물 확인

**리스크 오더 확인**: 리스크 관리 영역 → 유형별 탭 → 상태 카드 클릭 → 오더 조회 (유형+상태 필터 적용)

**참고**: 파트너 어드민에서는 오더 상태를 변경할 수 없습니다. 오더 발행/완료/취소는 인터널 어드민에서만 수행 가능합니다.
