# 오더 관리

## 1. 개요
- **화면명**: 오더 관리
- **목적**: 자사에 배정된 세차 오더를 조회하고, 미예약 오더에 수행원을 배정하여 예약을 진행
- **주요 기능**: 조회/필터링, 상세 확인 (Drawer), **오더 할당 (수행원 배정 + 예약)**
- **권한**: 조회 + 오더 할당 가능 — 오더 발행/완료/취소 등은 불가

### 인터널 오더 관리 대비 차이점
| 항목 | 인터널 (오더 관리) | 파트너 (오더 관리) |
| :--- | :--- | :--- |
| 화면명 | "오더 관리" | "오더 관리" |
| 데이터 범위 | 전체 오더 | 자사 배정 오더만 (`partner === currentPartner.partnerName`) |
| 오더 발행 | 수기 발행 Drawer 제공 | **제거** |
| 수행 완료 | 상세 Drawer에서 완료 처리 | **제거** |
| 오더 취소 | 상세 Drawer에서 취소 (사유 입력) | **제거** |
| **오더 할당** | 없음 | **추가** — 발행 상태 오더에 수행원 배정 + 예약 |
| 파트너 이름 컬럼 | 있음 | **제거** (단일 파트너) |
| 파트너 유형 컬럼 | 있음 | **제거** |
| 파트너 이름 필터 | 상세 필터 | **제거** |
| 파트너 유형 필터 | 상세 필터 | **제거** |
| 상세 파트너 유형 필드 | 기본 정보 카드에 표시 | **제거** |
| 리스트 컬럼 수 | 16개 | 14개 |
| 상세 필터 항목 수 | 10개 | 8개 |
| Drawer footer | 닫기 + 수행 완료 + 오더 취소 | 닫기 + **오더 할당** (발행 상태만) |
| 대시보드 연동 | quickFilter (인터널 대시보드) | initialFilter (파트너 대시보드) |

## 2. 데이터 정책

인터널 오더 관리의 데이터 정책(섹션 2.1~2.8)을 동일하게 참조합니다. 다음 사항은 파트너 어드민에서 **읽기 전용**으로 표시됩니다.

- 상태 전이 정책 (발행 → 예약 → 수행 중 → 완료, 취소)
- 취소 유형 및 책임 분류 (9종)
- 리드타임 및 지연 처리 (72h 기준)
- 파트너 유형별 정책 (현장/입고/핸들러)
- 점검 유형 매핑 (A/B/C/D)

## 3. 화면 구성 및 기능

### 3.1 헤더
| 요소 | 설명 |
| :--- | :--- |
| 타이틀 | "오더 관리" |
| 서브타이틀 | "{partnerName}에 배정된 오더를 조회하고 관리합니다." |
| **[목록 다운로드]** | 현재 필터 결과를 Excel 다운로드 |

- 인터널의 **[오더 발행]** 버튼은 제거

### 3.2 리스트

#### 검색 및 필터

**기본 필터** (항상 1줄 노출)

| 필드 | 타입 | 설명 |
| :--- | :--- | :--- |
| 차량 번호 | 텍스트 검색 | 검색 아이콘 포함. 차량 번호 기준 검색 |
| 세차 유형 | 셀렉트 | 내외부, 내부, 외부, 특수, 협의, 라이트, 기계세차 |
| 진행 상태 | 셀렉트 | 발행, 예약, 수행 중, 완료, 취소 |
| 발행 일시 | 날짜 범위 | 시작일~종료일. 기본값: 최근 1개월 |

**상세 필터** ("+ 필터 추가" 버튼으로 온디맨드 추가, 8개)

| 필드 | 타입 | 설명 |
| :--- | :--- | :--- |
| 차종 | SearchableSelect | 차량 모델명 |
| 존 이름 | SearchableSelect | 배정 존 |
| 오더 구분 | 셀렉트 | 긴급, 정규, 변경, 수시, 특별 |
| 발행 유형 | 셀렉트 | 위생장애, 고객 피드백(ML)_긴급, 초장기 미세차, 주기세차 등 |
| 지역1 | 셀렉트 | 시/도 |
| 지역2 | 셀렉트 | 구/군 (지역1 선택 전 비활성화. 지역1 변경 시 초기화) |
| 취소 유형 | 셀렉트 | 진행 상태 '취소' 선택 시 자동 추가. 취소 외 비활성화 |
| 지연 여부 | 셀렉트 | 진행 상태 '완료' 선택 시 자동 추가. 완료 외 비활성화. 지연/정상 |

- 인터널 대비 **파트너 이름**, **파트너 유형** 필터 제거

**필터 UI 동작**

- **필터 추가 플로우**: "+ 필터 추가" 클릭 → 드롭다운에서 항목 선택 → 상세 필터 영역에 추가 & 자동 펼침. 이미 추가된 필터는 드롭다운에서 제외
- **상세 필터 토글**: 1개 이상 추가 시 "상세 필터(N)" 토글 표시. 접기/펼치기 가능
- **자동 추가**: 진행 상태 '취소' 선택 시 취소 유형 자동 추가, '완료' 선택 시 지연 여부 자동 추가
- **개별 제거**: 상세 필터 항목에 마우스 호버 시 X 버튼으로 제거 가능
- **적용 필터 칩**: 값이 설정된 필터를 칩 형태로 표시. 각 칩 X 버튼으로 개별 해제. 2개 이상일 때 "전체 해제" 링크 표시
- **대시보드 필터 칩**: 대시보드에서 연동된 필터는 "대시보드 필터" 라벨로 칩 표시
- **초기화** (↻): 모든 필터 초기값 리셋 + 상세 필터 모두 제거 + 기간 최근 1개월 복원

#### 컬럼 (14개)
오더 ID, 오더 구분, 발행 유형, 세차 유형, 차량 ID, 차량 번호, 차종, 지역1, 지역2, 존 이름, 진행 상태(배지), 취소 유형(취소 시만), 지연 여부(완료 시만), 발행 일시

- 인터널 대비 **파트너 이름**, **파트너 유형** 컬럼 제거
- **상태 배지**: 완료(초록), 발행/예약(주황), 수행 중(파랑), 취소(회색)
- **지연 여부 배지**: 완료 상태일 때만 표시. 지연(빨강), 정상(초록). 그 외 상태는 `-` 표시
- **데이터 건수 표시**: 필터 패널 아래, 리스트 위에 표시. 필터 적용 시 `필터된 결과 N건 / 전체 **M**건`, 필터 미적용 시 `전체 **M**건`
- **기본 정렬**: 발행 일시 내림차순
- **페이지네이션**: 40건

#### 대시보드 연동 (initialFilter)
파트너 대시보드에서 `initialFilter` props로 필터값 전달 (status, orderType, cancelType, delayed)

### 3.3 상세 (Drawer)

3개의 탭으로 구성됩니다. 핸들러 파트너는 "미션 및 분실물" 탭 숨김.

- Drawer footer:
  - **닫기** 버튼 (항상 표시)
  - **오더 할당** 버튼 (진행 상태가 "발행"인 경우에만 표시)

#### ① 상세정보 탭

- **헤더**: 차량 번호(차종) + 진행 상태 배지
- **오더 정보 카드**: 오더 ID, 발행 일시, 수행 일시, 취소 일시 / 오더 구분, 발행 유형, 세차 유형 / 담당 수행원 / 진행 상태, 취소 유형(취소 시만) / 메모
- **차량 정보 카드**: 차량 ID, 차량 번호/차종, 존 이름, 지역
- **오더 체인 & 리드타임** (완료 오더만): 별도 카드
  - 참조 오더가 있는 경우: 세로 타임라인으로 취소→재발행 체인 시각화
    - 각 노드: 참조 오더 ID, 취소 유형(배지), 귀책/면책 표시, 발행·취소 시각 및 경과시간
    - 마지막 노드: 현재 오더(완료 배지), 재발행·완료 시각
  - 참조 오더가 없는 경우: "직접 완료 (참조 오더 없음)" 표시
  - 산출 요약 박스: 리드타임, 면책 시간, 귀책 소요시간, 판정(지연/정상). 지연 시 빨간 배경
- **탁송 정보** (입고 전용): 입고/출고 예약 ID + 탁송 상태. 별도 카드.
- **미션핸들 예약 정보** (핸들러 전용): 미션핸들 예약 ID + 예약 상태. 별도 카드.
- **금액 및 연계 정보** (핸들러 제외): 최종 청구 금액, 연계 오더(입고 전용)

#### ② 사진 및 점검 탭
- **진행 이력**: 상태 변경 타임라인
- **사진**: 세차 전/후 사진, 기타 사진 (D유형은 세차 전 사진 미노출)
  - 핸들러: 세차 전 1~5장, 세차 후 7~10장, 세차 후기 1~100자
- **점검 리스트**: 유형별(A~D) 점검 결과
  - 이상 항목: 빨간색 '이상' 배지 + 증빙 사진
  - 타이어 관련: 2×2 Grid (FL/FR/RL/RR)

#### ③ 미션 및 분실물 탭
- **미션**: 연동된 미션 목록 및 상태 (읽기 전용)
- **분실물**: 분실물 ID, 접수일시, 구분, 처리상태, 상세 정보 (읽기 전용)

## 4. 오더 할당 기능

### 4.1 개요

- **목적**: 예약되지 않은 오더(발행 상태)에 수행원을 강제 배정하여 세차를 수행하도록 예약
- **사용 대상**: 리드타임이 길어지거나, 자동 예약이 되지 않는 오더
- **진입점**: 오더 상세 Drawer footer의 "오더 할당" 버튼 (발행 상태 오더만 표시)

### 4.2 할당 플로우 (3단계)

Drawer 내부에서 3단계 스텝 UI로 진행됩니다. Drawer 타이틀이 "오더 할당 — {현재 단계}" 로 변경됩니다.

#### Step 1: 수행원 선택

- **진행 단계 표시**: 1→2→3 스텝 인디케이터 (현재 단계 파란색, 완료 단계 초록색)
- **오더 요약 카드**: 차량 번호(차종), 오더 ID, 세차 유형, 오더 구분, 상태 배지
- **존 담당 수행원 목록**: 해당 오더의 `zoneId`에 배정된 자사 수행원만 표시
  - 각 수행원 카드: 이름, ID, 담당존 수, 오늘 배정 건수 (색상 구분: 0~1건 초록, 2~3건 주황, 4건+ 빨강)
  - 패널티 있는 경우: 경고 아이콘 + "패널티 N회" 표시
  - 해당 존에 수행원이 없는 경우: 안내 메시지 표시
- **클릭 시**: 해당 수행원 선택 → Step 2로 이동

#### Step 2: 시간 선택

- **선택된 수행원 정보 카드**: 이름, ID, 오늘 배정 건수, 패널티 수
- **수행원 오늘 스케줄**: 해당 수행원의 오늘 배정 오더 리스트
  - 각 항목: 시간, 존 이름, 차량 번호, 세차 유형 배지
  - 배정이 없으면 "오늘 배정된 오더가 없습니다" 표시
- **차량 예약 가능 시간**: 대상 차량의 이용 가능 시간대
  - 차량 번호(차종), 이용 가능 사유, 가능 시간 윈도우 (배지)
- **예약 시간 선택 그리드**: 08:00~22:00 범위의 1시간 30분 단위 시간대
  - 각 슬롯 상태:
    - **예약 가능** (초록 테두리): 수행원 일정 비어있고 + 차량 이용 가능
    - **예약 불가** (회색): 수행원 일정 충돌 또는 차량 이용 불가 (사유 표시)
  - 가능한 슬롯 클릭 시 → Step 3으로 이동

#### Step 3: 할당 확인

- **오더 정보 요약**: 오더 ID, 차량, 존, 세차 유형
- **할당 내용 요약** (파란색 강조 박스): 수행원 이름(ID), 예약 시간
- **안내 메시지**: "할당 후 오더 상태가 '발행' → '예약'으로 변경됩니다."
- **할당 확정 버튼**: Drawer footer에 표시

### 4.3 할당 완료

- 확정 후 Drawer 내부에 완료 화면 표시
  - 체크 아이콘 + "오더 할당 완료" 메시지
  - 변경 결과 카드: 오더 ID, 변경 상태(예약 배지), 수행원, 예약 시간
- 오더 상태가 "발행" → "예약"으로 변경되고, 담당 수행원 및 예약 시간 설정

### 4.4 Footer 버튼 동작

| 단계 | 좌측 버튼 | 우측 버튼 |
| :--- | :--- | :--- |
| Step 1 (수행원 선택) | 취소 (할당 플로우 종료) | - |
| Step 2 (시간 선택) | 이전 (Step 1로) | - (시간 선택 시 자동 이동) |
| Step 3 (할당 확인) | 이전 (Step 2로) | **할당 확정** (파란 버튼) |
| 완료 | **닫기** (Drawer 닫기) | - |

## 5. 운영 시나리오

**오더 현황 확인**: 파트너 대시보드 → 상태 카드 클릭 → 오더 관리 페이지 (필터 자동 적용)

**오더 상세 확인**: 리스트에서 행 클릭 → Drawer 오픈 → ① 기본 정보 확인 → ② 사진/점검 확인 → ③ 미션/분실물 확인

**오더 할당 (수행원 강제 배정)**: 발행 상태 오더 상세 → "오더 할당" 버튼 → Step 1: 존 담당 수행원 선택 → Step 2: 수행원 스케줄/차량 예약 확인 & 시간 선택 → Step 3: 할당 확인 → 확정

**리스크 오더 확인**: 리스크 관리 영역 → 유형별 탭 → 상태 카드 클릭 → 오더 관리 (유형+상태 필터 적용)

**참고**: 오더 발행/완료/취소는 인터널 어드민에서만 수행 가능합니다. 파트너 어드민에서는 오더 할당(수행원 배정 + 예약)만 가능합니다.
