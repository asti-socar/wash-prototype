# 미션 조회

## 1. 개요
- **화면명**: 미션 조회
- **목적**: 자사에 배정된 미션 정책 현황과 할당 차량의 수행 상태를 읽기 전용으로 조회
- **주요 기능**: 정책 리스트 조회/필터링, 정책 상세 확인, 할당 차량 검색 (Drawer)
- **권한**: 읽기 전용 — 정책 등록/수정/삭제, 차량 추가/삭제, 상태 변경 불가

### 인터널 미션 정책 관리 대비 차이점
| 항목 | 인터널 (미션 정책 관리) | 파트너 (미션 조회) |
| :--- | :--- | :--- |
| 화면명 | "미션 정책 관리" | "미션 조회" |
| 서브타이틀 | "정책 기반 미션 등록 및 대상 차량 관리" | "{partnerName}에 배정된 미션 정책 및 차량을 조회합니다." |
| 미션 정책 등록 | 등록 버튼 + 등록 Drawer | **제거** |
| 정책 상태 변경 | 상세 Drawer에서 활성/비활성 Select | **제거** (읽기 전용 Badge 표시) |
| 정책 삭제 | 상세 Drawer Footer에 삭제 버튼 | **제거** |
| 차량 일괄 추가 | Textarea + 추가 버튼 | **제거** |
| 차량 개별 삭제 | 대기 차량 삭제 버튼 (행별) | **제거** |
| 대기 차량 전체 삭제 | 전체 삭제 버튼 | **제거** |
| 할당 차량 검색 | Textarea 검색 | **유지** |
| 상세 정책 정보 표시 | Input (disabled) | 읽기 전용 텍스트 블록 |
| Drawer Footer | 정책 삭제 + 닫기 | 닫기만 |

## 2. 데이터 정책

인터널 미션 정책 관리의 데이터 정책을 동일하게 참조합니다. 파트너 어드민에서는 **읽기 전용**으로 표시됩니다.

- 오더 연동 정책 (자동 포함, 선택적 수행, 상태 동기화)
- 비활성 전환 시 기발행 오더 처리 정책

## 3. 화면 구성 및 기능

### 3.1 헤더
| 요소 | 설명 |
| :--- | :--- |
| 타이틀 | "미션 조회" |
| 서브타이틀 | "{partnerName}에 배정된 미션 정책 및 차량을 조회합니다." |

- 인터널의 **[미션 정책 등록]** 버튼은 제거

### 3.2 검색 및 필터

상단 카드 영역에서 정책 리스트를 필터링하며, 선택된 조건은 하단에 **칩(Chip)** 형태로 노출됩니다.

#### 검색 조건
| 구분 | 항목 | 타입 | 동작 정의 |
| :--- | :--- | :--- | :--- |
| **검색** | 미션 제목 | 텍스트 검색 | 검색 아이콘 포함. 부분 일치 검색 |
| **기간** | 등록 일시 | 날짜 범위 | 시작일 `~` 종료일 한 줄 배치 |
| **필터** | 상태 | 선택(Select) | 전체, 활성, 비활성 |

#### UI 동작
- **필터 칩**: 적용된 조건이 칩으로 나열, `X` 버튼으로 개별 해제
- **설정 초기화**: 모든 입력값 및 칩 제거

### 3.3 미션 정책 목록

#### 컬럼 (8개)
| 헤더명 | 데이터 정의 |
| :--- | :--- |
| ID | 정책 고유 식별자 |
| 미션 제목 | `title` 필드 (없을 경우 `content`) |
| 금액 | 3자리 콤마 포맷 (`{금액}원`) |
| 대상 차량수 | 할당된 전체 차량 수 (`{N}대`) |
| 수행 완료수 | 'completed' 상태 차량 수 (`{N}대`) |
| 진행률(%) | `(수행 완료수 / 대상 차량수) * 100` (반올림) |
| 상태 | 배지 — 활성(Green), 비활성(Default) |
| 등록일시 | `yyyy-MM-dd hh:mm` |

- **기본 정렬**: ID 내림차순
- **페이지네이션**: 40건

### 3.4 정책 상세 (Drawer) — 읽기 전용

목록의 행 클릭 시 Drawer 오픈. Footer에는 **닫기** 버튼만 표시.

#### A. 정책 정보 카드
- **미션 제목**: 읽기 전용 텍스트 블록
- **미션 내용**: 읽기 전용 텍스트 블록 (whitespace-pre-wrap)
- **미션 금액**: 읽기 전용 텍스트 (`{금액}원`)
- **상태**: 배지 표시 (활성/비활성) — 인터널과 달리 **변경 불가**
- **등록일시**: 텍스트 표시
- **증빙 사진 필수**: 체크박스 (disabled)

#### B. 할당 차량 조회 카드

**차량 번호 검색**
- Textarea에 차량 번호를 콤마 또는 줄바꿈으로 구분하여 입력
- 입력된 번호와 일치하는 차량만 아래 목록에 필터링 표시

**할당된 차량 목록**
- **헤더**: "할당된 차량 목록 ({완료 차량수} / {할당 차량 총수}대)" + 진행률 배지
- **페이지네이션**: 100건

| 컬럼 | 설명 |
| :--- | :--- |
| 차량 번호 | 할당 차량 번호 |
| 수행 상태 | 배지 — 완료(Green), 대기(Orange) |
| 연결된 오더 | 완료 시 오더 ID 표시, 미완료 시 `-` |
| 완료일 | `YYYY-MM-DD` 형식, 미완료 시 `-` |

- 인터널의 **관리(삭제 버튼)** 컬럼 제거
- 인터널의 **차량 일괄 추가** 영역 제거
- 인터널의 **대기 차량 전체 삭제** 버튼 제거

---

## 4. 운영 시나리오

**미션 정책 현황 조회**: 미션 조회 진입 → 필터로 활성/비활성 구분 → 진행률 확인

**특정 차량 수행 여부 확인**: 정책 행 클릭 → Drawer 오픈 → 차량 번호 검색 Textarea에 차량 번호 입력 → 해당 차량의 수행 상태(완료/대기) 및 연결된 오더 확인

**참고**: 미션 정책 등록/수정/삭제 및 차량 추가/삭제는 인터널 어드민에서만 수행 가능합니다. 파트너 어드민에서는 조회만 가능합니다.
