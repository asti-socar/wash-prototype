# 파트너 관리

## 1. 개요
- **화면명**: 파트너 관리
- **목적**: 세차 파트너사의 기본 정보, 연락처, 사업자 정보 및 단가 정책을 관리
- **주요 기능**: 파트너 목록 조회, 신규 등록, 정보 수정, 삭제(soft delete), 단가 정책 관리

## 2. 화면 구성 및 기능

### 2.1 리스트
- **[신규 파트너 등록]** 버튼으로 등록 Drawer 호출
- **페이지네이션**: 40건
- **기본 정렬**: 파트너 ID 내림차순

#### 컬럼
| 헤더명 | 비고 |
| :--- | :--- |
| 파트너 ID | |
| 구분 | 파트너 구분 |
| 하위 구분 | 파트너 하위 구분 (입고 세차장 / 현장 세차장) |
| 파트너 이름 | |
| 주소 | 기본 정보 주소 + 상세 주소 |
| 법인명 | |

### 2.2 상세/등록 (Drawer)
행 클릭 시 상세 조회. 2개 탭으로 구성.

#### ① 파트너 정보 탭

모든 입력 필드는 필수입니다.

##### 카드 1: 기본 정보
| 필드 | 타입 | 비고 |
| :--- | :--- | :--- |
| 파트너 구분 | Input (읽기 전용) | 고정값: "세차 파트너" |
| 파트너 하위 구분 | Select | 입고 세차장 / 현장 세차장 |
| 파트너 이름 | Input | |
| 주소 | Input(readOnly) + 우편번호/지도 검색 | 우편번호 검색 또는 지도 검색 모드 선택. 주소 Input은 항상 readOnly (API 응답값만 입력) |
| 상세 주소 | Input | |

##### 카드 2: 주 연락처
| 필드 | 타입 | 비고 |
| :--- | :--- | :--- |
| 이름 | Input | |
| 전화번호 | Input | |

##### 카드 3: 사업자 정보
| 필드 | 타입 | 비고 |
| :--- | :--- | :--- |
| 사업자등록번호 | Input | |
| 법인명 | Input | |
| 대표자 | Input | |
| 업태 | Input | |
| 업종 | Input | |
| 주소 | Input(readOnly) + 우편번호/지도 검색 | 기본 정보 주소와 별도. 우편번호 검색 또는 지도 검색 |
| 상세 주소 | Input | |

- 주소 필드: 라디오 버튼으로 "우편번호 검색" / "지도 검색" 모드 전환. 두 모드 모두 주소 Input은 항상 readOnly (API 응답값만 입력).
  - 우편번호 검색: Daum Postcode API 연동
  - 지도 검색: Naver Maps JavaScript API v3 (submodules=geocoder). 검색바로 Geocoding(주소→좌표), 지도 클릭으로 Reverse Geocoding(좌표→주소, ROAD_ADDR 우선). API 로드 실패 시 amber 경고 카드 표시.
- Footer: 좌측 [삭제] (수정 모드만) + 우측 [닫기] + [수정하기]/[등록하기]
- 삭제: confirm 후 soft delete 처리

#### ② 단가 정책 관리 탭
파트너별 오더 구분/세차 유형 조합에 따른 단가를 관리합니다.

**적용 중 단가 정책**: 적용 시작일이 오늘 이전인 정책 (최신순 정렬)

**예정된 단가 정책**: 적용 시작일이 오늘 이후인 정책 (날짜순 정렬)

**정책 추가 폼**:
- 오더 구분 (긴급/정규/변경/수시/특별)
- 세차 유형 (내외부/내부/외부/특수/협의/라이트/기계세차)
- 단가 (숫자)
- 적용 시작일 (DatePicker)
- confirm 후 추가

**정책 삭제**: 각 행의 삭제 아이콘 클릭 → confirm 후 삭제

- Footer: 좌측 [삭제] (수정 모드만) + 우측 [닫기]
