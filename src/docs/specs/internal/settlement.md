# 합의 요청 관리

## 1. 개요
- **화면명**: 합의 요청 관리
- **목적**: 세차 수행 중 발견된 특이사항으로 파트너사가 추가 비용을 요청할 경우, 검토 및 합의(승인/반려/네고) 프로세스를 관리
- **주요 기능**: 요청 목록 조회, 사유 및 현장 사진 확인, 청구 금액 조정, 승인/반려 처리

## 2. 데이터 정책

### 2.1 상태 전이 및 수정 제한

#### 승인 프로세스별 상태 흐름

| 프로세스 | 상태 흐름 | 청구 금액 수정 | 비고 |
| :--- | :--- | :--- | :--- |
| **자동 승인** | → `승인` (즉시) | 인터널: 불가 / 파트너: 불가 | 현장 변경 전용. 사후 반려 가능 (`승인` → `반려`) |
| **1단계 승인** | `요청` → `승인` / `반려` | 인터널: 불가 / 파트너: `요청` 상태에서만 가능 | 파트너 또는 인터널 선착순 1인 처리 |
| **2단계 승인** | `요청` → 파트너 1차 승인 → `승인` / `반려` | 인터널: 불가 / 파트너: `요청` 상태에서만 가능 | 파트너 선승인 후 인터널 최종 처리 |

- **종결 처리**: `반려`는 최종 상태, 이후 변경 불가
- **사후 반려 (자동 승인 한정)**: 자동 승인된 `승인` 상태에서 인터널이 수행 결과 확인 후 `반려` 전환 가능. 반려 시 청구 관리에서 미지불 건 처리

### 2.2 승인 프로세스 유형

| 구분 | 자동 승인 | 1단계 승인 | 2단계 승인 |
| :--- | :--- | :--- | :--- |
| 노출 방식 | 인터널에만 결과 노출 | 파트너 + 인터널 동시 노출 | 파트너 선노출 → 선승인 후 인터널 노출 |
| 승인 주체 | 시스템 (자동) | 파트너 또는 인터널 (선착순 1인) | 파트너(1차) → 인터널(최종) |
| 목적 | 현장 세차 유형 변경 즉시 반영 | 현장 대기 시간 최소화 | 고비용 건 비용 남용 방지 |

#### 2단계 승인 일시 정책 (인터널 관점)
- **요청 일시**: 1차 처리자(파트너)가 선승인을 완료한 시점
- **처리 일시**: 2차 처리자(인터널)가 최종 승인/반려한 시점

> 1단계 승인의 경우, 요청 일시는 수행원의 합의 요청 접수 시점, 처리 일시는 처리한 시점입니다.

### 2.3 처리 주체 판별
| 처리자 유형 | 표기 | 판별 기준 |
| :--- | :--- | :--- |
| 내부 관리자 | 인터널 | 영문 닉네임 (예: brown, asti) |
| 파트너 관리자 | 파트너 | 한글 이름 (예: 김길동) |

> 2단계 승인에서는 인터널만 처리 가능하므로 항상 "인터널" 표시

### 2.4 요청 유형 (4개 대분류)

| 요청 유형 | 승인 프로세스 | 설명 |
| :--- | :--- | :--- |
| **현장 변경** | 자동 승인 | 현장 세차 내에서 세차유형만 변경. 자동 승인되나 사후 반려 가능 |
| **전환** | 1단계 (현장→입고) / 2단계 (현장→핸들러) | 현장에서 입고 또는 핸들러(기계세차)로 파트너 유형 전환 |
| **입고 변경** | 2단계 | 입고 세차 내에서 세차유형 변경 |
| **추가 미션** | 2단계 | 세차 외 추가 작업 (예: 대형 폐기물 처리) |

#### 현장 변경 (자동 승인)
- 발생 즉시 `승인` 상태로 자동 처리
- 인터널이 수행 결과 확인 후 **사후 반려** 가능 (승인 → 반려)
- 반려 시 청구 관리에서 미지불 건으로 처리

| # | 변경 전 | 변경 후 |
| :---: | :--- | :--- |
| 1 | 라이트 | 내부 |
| 2 | 라이트 | 외부 |
| 3 | 라이트 | 내외부 |
| 4 | 내부 | 내외부 |
| 5 | 외부 | 내외부 |

#### 전환
현장 세차에서 입고 또는 핸들러(기계세차)로 파트너 유형이 전환되는 케이스.
- **현장→입고**: 1단계 승인
- **현장→핸들러**: 2단계 승인

| # | 변경 전 | 변경 후 | 승인 |
| :---: | :--- | :--- | :---: |
| 1 | 현장: 라이트 | 입고: 특수 | 1단계 |
| 2 | 현장: 내부 | 입고: 특수 | 1단계 |
| 3 | 현장: 외부 | 입고: 특수 | 1단계 |
| 4 | 현장: 내외부 | 입고: 특수 | 1단계 |
| 5 | 현장: 라이트 | 핸들러: 기계세차 | 2단계 |
| 6 | 현장: 내부 | 핸들러: 기계세차 | 2단계 |
| 7 | 현장: 외부 | 핸들러: 기계세차 | 2단계 |
| 8 | 현장: 내외부 | 핸들러: 기계세차 | 2단계 |

#### 입고 변경 (2단계 승인)

| # | 변경 전 | 변경 후 |
| :---: | :--- | :--- |
| 1 | 내외부 | 특수 |
| 2 | 내외부 | 협의 |

#### 추가 미션 (2단계 승인)
세차 외 추가 작업이 필요한 케이스. 변경 전/후 세차유형 해당 없음 (`-` 표시).

### 3.2 상세 (Drawer)

- **타이틀**: `합의 요청 상세 - {요청ID}`

#### A. 요청 정보 섹션
- 오더 ID, 차량 번호, 차종, 존 이름, 파트너 이름, 요청 유형, 요청 일시
- 변경 전 세차유형, 변경 후 세차유형 (추가 미션은 `-`)
- 합의 유형 (자동 승인 / 1단계 / 2단계)
- 상태 (요청 / 승인 / 반려)
- **자동 승인(현장 변경)**: 처리 주체 `시스템`, 처리 일시 = 요청 일시. 사후 반려 버튼 노출
- **1단계**: 처리 주체 (인터널 / 파트너), 처리자, 처리 일시
- **2단계**: 1차 처리자 (파트너 담당자), 2차 처리자 (인터널 담당자), 처리 일시

#### 금액 및 코멘트
| 필드 | 설명 |
| :--- | :--- |
| 청구 금액 | 인터널 어드민에서는 항상 읽기 전용. 파트너 어드민에서만 `요청` 상태일 때 수정 가능 |
| 요청 코멘트 | 수행원이 입력한 합의 요청 메시지 |
| 반려 코멘트 | `반려` 상태에서만 표시 |

#### C. 승인 및 반려 처리 (Footer)
- `요청` 상태 또는 사후 반려 가능 시 액션 버튼 노출, 그 외 `닫기`만 제공

##### 승인
- [승인] 클릭 시 확인 다이얼로그 표시: "승인 후 청구금액 수정이 불가능합니다. 승인하시겠습니까?"
- 확인 → 상태 `승인`, 처리자·처리 일시 기록 후 드로어 닫힘
- 취소 → 아무 동작 없음

##### 반려 (2단계 모달 플로우)
1. **사유 입력 모달**: [반려] 클릭 시 모달 팝업으로 반려 사유 입력 폼 표시
   - 반려 사유 필수 입력 (미입력 시 [반려 확정] 비활성화)
   - [반려 확정] → 확인 모달로 이동
   - [취소] → 모달 닫힘
2. **최종 확인 모달**: 요청 유형, 변경 전/후, 청구 금액, 반려 사유 요약 표시
   - [반려 처리] → 상태 `반려` + 반려 코멘트 저장, 드로어 유지 (갱신된 데이터 표시)
   - [이전] → 사유 입력 모달로 돌아감

##### 사후 반려 (현장 변경 전용)
- 자동 승인된 `승인` 상태에서 [사후 반려] 버튼 노출
- 클릭 시 사유 입력 모달에 경고 안내 배너 표시: "자동 승인된 건을 사후 반려합니다. 반려 시 청구 관리에서 미지불 건으로 처리됩니다."
- 이후 반려와 동일한 2단계 모달 플로우 진행
