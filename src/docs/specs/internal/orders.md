# 오더 관리

## 1. 개요
- **화면명**: 오더 관리
- **목적**: 세차 오더의 생명주기(발행~완료) 관리 및 모니터링
- **주요 기능**: 조회/필터링, 수기 발행, 상세/타임라인 확인, 상태 변경(완료/삭제)

## 2. 데이터 정책

### 2.1 데이터 모델
| 구분 | 필드 구성 | 비고 |
| :--- | :--- | :--- |
| **오더 기본** | ID, 구분(긴급/정규/변경/수시/특별), 발행유형, 세차유형, 점검유형(A/B/C/D), 상태, 취소유형, 참조 오더 ID | 점검유형은 세차유형에 종속. 취소유형은 취소 상태에서만 유효 |
| **차량 정보** | 차량 ID, 번호, 차종, 존(ID/이름), 지역(1/2) | 발행 시 실시간 조회 |
| **파트너** | 파트너 이름, 유형(현장/입고/핸들러), 수행원 | 유형 기본값: 현장 |
| **탁송 정보** | 입고/출고 예약 ID, 탁송 상태 | 입고 파트너 전용. 출고는 완료 상태에서만 표시 |
| **상세 데이터** | 메모, 사진(전/후/기타), 점검결과, 미션(스냅샷), 분실물 | - |

### 2.2 상태 전이
- **기본 흐름**: `발행` → `예약` → `수행 중` → `완료` (모든 단계에서 `취소` 가능)
- **탁송 상태** (핸들러): WAITING → ASSIGN → RUN → END/FORCE_END/CANCEL

### 2.3 취소 유형

| # | 취소 유형 | 책임 | 설명 | 취소 주체 | 취소 대상 |
| :---: | :--- | :---: | :--- | :---: | :--- |
| 1 | 변경 | 면책 | 오더 내용 변경 시 기존 오더 취소 후 재발행 | 시스템 | 오더/예약 취소 |
| 2 | 미예약 | 귀책 | 오더 발행 후 24시간 내 파트너 예약 미완료 | 시스템 | 오더 취소 |
| 3 | 예약 불가 | 면책 | 미예약 취소 조건에서, 수행원이 예약할 수 없었던 것으로 판단되는 경우 | 시스템 | 오더 취소 |
| 4 | 노쇼 | 귀책 | 수행원이 예약 시간에 맞춰 시작하지 않은 경우 | 시스템 | 예약 취소 |
| 5 | 우천 | 면책 | 우천으로 세차 불가 | 시스템 | 예약 취소 |
| 6 | 차량 없음 | 면책 | 예약한 차량이 존에 없음 | 수행원 | 예약 취소 |
| 7 | 주차장 문제 | 면책 | 주차장 문제로 세차 불가 | 수행원 | 예약 취소 |
| 8 | 기타 사유 | 면책 | 그외 피치 못할 사유 | 수행원 | 예약 취소 |
| 9 | 개인 사유 | 귀책 | 수행원 개인 사정 | 수행원 | 예약 취소 |

#### 취소 유형별 정책 상세

> **1. 변경 취소**
> - 새로 발행된 오더의 우선순위가 높을 경우, 파트너사가 바뀌어야 하는 경우에는 오더를 취소하고 재발행
> - 파트너사 변경이 안 되는 경우에는 오더 구분, 발행 유형, 세차 유형만 변경
> - 예약이 있는 경우에는 세차 수행 시작 전까지 변경 가능
> - 참조 오더 ID 불필요 (변경 취소로 재발행된 오더는 이전 오더를 참조하지 않음)

> **3. 예약 불가 판단 조건**
> - **현장 세차**: `세차수행시간 + 4시간`이 해당 차량 예약 테이블에 예약 가능 시간으로 존재하지 않는 경우
> - **입고 세차**: `배반차 핸들시간 + 쿠션시간 + 세차수행시간 + 4시간`이 해당 차량 예약 테이블에 예약 가능 시간으로 존재하지 않는 경우
> - **예약 가능 시간**: (1) 예약이 없는 시간 구간 또는 (2) 장애블락 예약 구간 (수행원이 세차블락으로 변경 가능)
> - 취소하는 시점 기준으로 예약 가능 여부를 판단

> **4. 노쇼 취소**
> - 세차 예약 시간으로부터 30분 동안 문열기, 수행 시작 등 주요 버튼을 누르지 않을 경우 예약 취소
> - 세차 시간은 총 50분, 이 중 최소 20분의 실제 수행 시간이 확보되어야 하기 때문에 30분 후 자동 취소 프로세스 적용

> **5. 우천 취소**
> - 세차 수행 4시간 전의 일기예보가 강수확률 60% 이상이며, 시간당 강수량 5mm 이상일 때 세차가 불가한 상황으로 판단

> **8. 기타 사유**
> - 일기예보가 잡아내지 못한 우천 상황 등

### 2.4 오더 참조 관계
오더는 미수행(취소) 시 동일 차량에 대해 재생성될 수 있으며, 재생성된 오더는 이전 오더를 참조합니다.

- **참조 오더 ID**: 직전에 취소된 오더의 ID (없으면 최초 발행 오더)
- **참조 오더 발행일시**: 직전에 취소된 오더의 발행 시점
- 참조 관계는 연쇄적으로 발생할 수 있음 (A → B → C → …)
- **최초 발행 오더 식별**: 참조 오더 ID가 없는 오더가 최초 발행 오더

### 2.5 리드타임 및 지연 처리

#### 용어 정의
| 용어 | 정의 | 산출 |
| :--- | :--- | :--- |
| **리드타임** | 최초 오더 발행 ~ 세차 완료 총 소요시간 | 참조 오더 체인 전체 구간 합산 |
| **면책 시간** | 수행원 귀책이 아닌 사유로 인정되는 시간 | 취소 유형의 책임="면책"인 취소~재발행 구간의 합 |
| **귀책 소요시간** | 수행원 책임으로 산정되는 실질 소요시간 | 리드타임 − 면책 시간 |
| **지연 처리** | 귀책 소요시간이 72시간을 초과한 오더 | 귀책 소요시간 > 72h |

#### 산출 규칙
- **리드타임**: 참조 관계(취소→재발행 체인)로 연결된 모든 오더의 구간을 합산한 총 소요시간
- **면책 시간**: 취소 유형의 책임이 "면책"(변경, 예약 불가, 우천, 차량 없음, 주차장 문제, 기타 사유)인 취소 발생 시, 해당 취소~재발행 사이 경과 시간의 합
- **귀책 소요시간**: 리드타임에서 면책 시간을 차감한 값. 수행원이 실질적으로 책임지는 소요시간
- **지연 처리**: 귀책 소요시간 > 72h인 완료 오더에 대해 발동. 해당 수행원에게 벌점 부과

#### 산출 예시

```
[오더A] 발행 → 2h 후 예약 → 세차시작 예정 15h 후
        → 세차시작 4h 전 우천 취소(면책) → 4h 대기 후 재발행
[오더B] 재발행 → 24h 미예약 취소(귀책) → 즉시 재발행
[오더C] 재발행 → 3h 후 예약 → 24h 후 세차시작 → 1h 세차 → 완료

리드타임     = 2 + (15−4) + 4 + 24 + 3 + 24 + 1 = 69h
면책 시간    = 4h (우천 대기)
귀책 소요시간 = 69 − 4 = 65h
→ 65h < 72h → 정상 완료 (지연 아님)
```

### 2.6 벌점

벌점은 **수행원 단위**로 부과되며, 다음 두 유형의 건수를 합산합니다.

| 벌점 유형 | 조건 | 부과 대상 | 배지 |
| :--- | :--- | :--- | :--- |
| 미예약 | 오더 발행 후 24시간 내 예약 미완료로 자동 취소 | 담당 존 파트너 소속 **전체 수행원** | warn (주황) |
| 노쇼 | 수행원이 예약 시간에 맞춰 수행을 시작하지 않음 | 해당 수행원 | danger (빨강) |

- **벌점 합계** = 미예약 취소 건수 + 노쇼 취소 건수
- 미예약 벌점은 해당 오더의 담당 존에 배정된 파트너 소속 **모든 수행원**에게 일괄 부과
- **매월 1일 00:00** 전체 수행원 벌점 자동 초기화 (월 단위 집계)
- 수행원 조회 화면에서 벌점 합계 및 이력 확인 가능

### 2.6 파트너 유형별 정책
| 파트너 유형 | 허용 세차 유형 | UI 특이사항 |
| :--- | :--- | :--- |
| **현장** | 내외부, 내부, 외부, 라이트 | 기본 흐름 |
| **입고** | 내외부, 협의, 특수 | 탁송 정보 카드 표시 (기본 정보 → 탁송 정보 → 금액/연계) |
| **핸들러** | 기계세차 | "미션 및 분실물" 탭 숨김, 수행원 `-`, 금액 카드 숨김, 미션핸들 예약 정보 카드 표시 |

### 2.7 점검 유형 매핑
세차 유형 선택 시 점검 유형이 자동 결정됩니다.
- **A유형**: 입고(특수, 협의) — 전체 점검 + 긴급조치/시트탈거
- **B유형**: 입고(내외부), 현장(긴급) — 전체 점검 (긴급조치 제외)
- **C유형**: 현장(내외부, 내부, 외부) — 해당 영역 항목만
- **D유형**: 라이트 — 라이트 전용 항목

### 2.8 미션 연동
- **발행 시**: 차량의 `대기` 미션 → 오더에 스냅샷 저장
- **수행 시**: 수행원이 선택적 처리 (강제 아님), 완료된 미션만 `완료` 상태로 변경
- **취소 시**: 오더 연결 해제, 수행 완료 미션은 `완료` 유지, 미수행 미션은 `대기` 복원

**필터 UI 동작**

- **필터 추가 플로우**: "+ 필터 추가" 클릭 → 드롭다운에서 항목 선택 → 상세 필터 영역에 추가 & 자동 펼침. 이미 추가된 필터는 드롭다운에서 제외
- **상세 필터 토글**: 1개 이상 추가 시 "상세 필터(N)" 토글 표시. 접기/펼치기 가능
- **자동 추가**: 진행 상태 '취소' 선택 시 취소 유형 자동 추가, '완료' 선택 시 지연 여부 자동 추가
- **개별 제거**: 상세 필터 항목에 마우스 호버 시 X 버튼으로 제거 가능
- **적용 필터 칩**: 값이 설정된 필터를 칩 형태로 표시. 각 칩 X 버튼으로 개별 해제. 2개 이상일 때 "전체 해제" 링크 표시
- **초기화** (↻): 모든 필터 초기값 리셋 + 상세 필터 모두 제거 + 기간 최근 1개월 복원

#### 대시보드 연동 (Quick Filter)
대시보드에서 `quickFilter` props로 필터값 전달 (status, orderType, cancelType, partner)

### 3.2 상세 (Drawer)

3개의 탭으로 구성됩니다. 핸들러 파트너는 "미션 및 분실물" 탭 숨김.

#### ① 상세정보 탭
- **기본 정보**: 오더 ID, 차량 ID, 차량 번호, 차종, 존, 지역, 발행/세차 유형, 파트너, 수행원, 진행 상태, 취소 유형(취소 시만, 진행 상태 우측), 발행/수행 일시, 메모
- **오더 체인 & 리드타임** (완료 오더만): 별도 카드
  - 참조 오더가 있는 경우: 세로 타임라인으로 취소→재발행 체인 시각화
    - 각 노드: 참조 오더 ID, 취소 유형(배지), 귀책/면책 표시, 발행·취소 시각 및 경과시간
    - 마지막 노드: 현재 오더(완료 배지), 재발행·완료 시각
  - 참조 오더가 없는 경우: "직접 완료 (참조 오더 없음)" 표시
  - 산출 요약 박스: 리드타임, 면책 시간, 귀책 소요시간, 판정(지연/정상). 지연 시 빨간 배경
- **탁송 정보** (입고 전용): 입고/출고 예약 ID + 탁송 상태. 별도 카드.
- **미션핸들 예약 정보** (핸들러 전용): 미션핸들 예약 ID + 예약 상태. 별도 카드.
- **금액 및 연계 정보** (핸들러 제외): 최종 청구 금액, 연계 오더(입고 전용)
- **[수행 완료]**: 오더 상태 "완료" 처리
- **[오더 취소]**: 취소 확인 팝업(모달) 표시. 취소 상태에서는 버튼 숨김
  - 취소 유형: Select (9종 — 변경, 미예약, 노쇼, 예약 불가, 우천, 차량 없음, 주차장 문제, 기타, 개인 사유)
  - 상세 사유: 텍스트 입력 (선택)
  - 취소 유형 입력 시 [취소 확정] 버튼 활성화
  - 확정 시 해당 오더의 상태를 '취소'로, 취소 유형·취소 일시 자동 기록

#### ② 사진 및 점검 탭
- **진행 이력**: 상태 변경 타임라인
- **사진**: 세차 전/후 사진, 기타 사진 (D유형은 세차 전 사진 미노출)
  - 핸들러: 세차 전 1~5장, 세차 후 7~10장, 세차 후기 1~100자
- **점검 리스트**: 유형별(A~D) 점검 결과
  - 이상 항목: 빨간색 '이상' 배지 + 증빙 사진
  - 타이어 관련: 2×2 Grid (FL/FR/RL/RR)

#### ③ 미션 및 분실물 탭
- **미션**: 연동된 미션 목록 및 상태
- **분실물**: 분실물 ID(클릭 시 새창), 접수일시, 구분, 처리상태, 상세 정보

### 3.3 오더 수기 발행 (Drawer)
1. 차량번호 입력 → 차량정보 자동 조회 (차종, 존, 파트너 표시)
2. 오더 구분, 발행유형, 세차유형 선택
3. [발행하기] → 발행하는 내용 재확인
- **필수값**: 차량번호(DB 존재 확인), 오더구분, 발행유형, 세차유형
- **차량 미등록 시**: 발행 차단 + 경고 메시지

## 4. 운영 시나리오

**긴급 오더 발행**: [오더 발행] → 차량번호 → 차량정보 확인 → "긴급" 선택 → 유형 선택 → [발행하기]

**오더 완료 처리**: 행 클릭 → ② 사진 및 점검 탭에서 확인 → ③ 미션 확인 → [수행 완료]

**예외사항**:
- 차량 조회 실패 시 오더 발행 불가 (차량 등록 선행 필요)
- 오더 취소 시 반드시 사유 입력 (감사 로그 기록)
- 미션은 수행원이 선택적으로 처리, 수행 완료한 미션만 '완료' 상태
