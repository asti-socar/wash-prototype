# 수행원 조회

## 1. 개요
- **화면명**: 수행원 조회
- **목적**: 현장 수행원의 배치 현황을 모니터링하고 품질 이슈 발생 시 피드백 근거로 활용
- **정책**: 인터널 어드민에서는 조회 전용 (편집 없음)

## 2. 데이터 정책

### 2.1 벌점 유형
벌점은 **수행원 단위**로 부과되며, 아래 두 유형의 건수를 합산합니다.

| 유형 | 조건 | 배지 |
| :--- | :--- | :--- |
| 지연 | 귀책 소요시간 > 72h인 완료 오더 | warn (주황) |
| 노쇼 | 취소 유형이 "노쇼"인 취소 건 | danger (빨강) |

> **용어 참고** (상세 정의: orders.md 섹션 2.5)
> - **리드타임**: 최초 오더 발행 ~ 세차 완료 총 소요시간
> - **면책 시간**: 취소 유형의 책임="면책"인 취소~재발행 구간의 합
> - **귀책 소요시간**: 리드타임 − 면책 시간 (수행원 실질 소요시간)
> - **지연 처리**: 귀책 소요시간 > 72h일 때 발동, 벌점 부과

## 3. 화면 구성 및 기능

### 3.1 리스트
- **필터**: 파트너 이름 Select
- **페이지네이션**: 40건
- **기본 정렬**: 수행원 ID 내림차순
- **정렬**: 정렬 가능 컬럼에 한해 헤더 클릭 시 오름차순/내림차순 토글. 정렬 가능 컬럼은 헤더에 화살표 아이콘(↕)으로 표시.

#### 컬럼
| 헤더명 | 정렬 | 비고 |
| :--- | :---: | :--- |
| 수행원 ID | | |
| 이름 | | |
| 파트너 이름 | | |
| 배정 존 개수 | | 배정된 쏘카존 수 |
| 벌점 | O | 0이면 회색 텍스트, 1 이상이면 빨간 배지 |

### 3.2 상세 (Drawer)

#### 기본 정보
수행원 ID, 이름, 파트너 이름 (모두 읽기 전용)

#### 배정된 쏘카존
- 배정 존 개수만큼 존 정보 리스트: `{존 이름} ({존 ID}) - {지역1}/{지역2}`
- 배정 존이 없으면 "배정된 쏘카존이 없습니다." 표시

#### 벌점 이력
- 벌점이 있는 경우: 오더 ID, 사유(지연/노쇼 Badge), 일자(`yyyy-MM-dd`) 테이블
- 벌점이 없는 경우: "벌점 이력이 없습니다." 표시
- 오더 ID 클릭 시 새 탭으로 오더 관리 상세 화면 이동
