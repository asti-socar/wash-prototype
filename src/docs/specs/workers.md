# 수행원 조회

## 1. 개요
- **목적**: 현장 수행원의 배치 현황을 모니터링하고 품질 이슈 발생 시 피드백 근거로 활용합니다.

## 2. 주요 기능
### 2.1 수행원 리스트 (View Only)
- **필터**: 파트너 이름, 지역1, 지역2.
  - 지역1 선택 전까지 지역2 필터는 비활성화 상태 (회색 처리)
  - 지역1 선택 시 지역2 활성화 및 해당 지역1에 속한 지역2만 옵션 표시
  - 지역1 변경 시 지역2 필터 자동 초기화
- **데이터 그리드**: 수행원 ID, 이름, 파트너 이름, 지역1, 지역2, 배정된 쏘카존, 벌점.

| 컬럼 | 설명 |
|:---|:---|
| 수행원 ID | 수행원 고유 ID |
| 이름 | 수행원 성명 |
| 파트너 이름 | 소속 파트너 이름 |
| 지역1 | 시/도 |
| 지역2 | 구/군 |
| 배정된 쏘카존 | 배정 존 이름 |
| 벌점 | 누적 벌점 (0이면 회색, 1 이상이면 빨간 배지) |

### 2.2 수행원 상세 (Drawer)
- **현황 파악**: 어느 회사 소속인지, 어떤 쏘카존들에 배정되어 있는지 상세 확인.
- **정책**: 인터널 어드민에서는 모니터링 목적의 '뷰잉'만 제공하며, 편집은 생략합니다.

#### 상세 화면 구성
| 영역 | 필드 |
|:---|:---|
| 기본 정보 | 수행원 ID, 이름, 파트너 이름 |
| 배정된 쏘카존 | 존 이름 (존 ID) |
| 벌점 이력 | 오더 ID, 사유, 일자 테이블 |

## 3. 벌점 정책

### 3.1 벌점 유형

| 유형 | 조건 | 배지 |
|:---|:---|:---|
| 지연 | 오더 발행 후 24시간 이내에 예약하지 않은 경우 | warn (주황) |
| 노쇼 | 예약한 오더에 정시 도착하지 않은 경우 | danger (빨강) |

### 3.2 벌점 이력 표시
- 벌점이 있는 경우: 오더 ID, 사유(지연/노쇼), 일자(yyyy-MM-dd) 를 테이블로 표기
- 벌점이 없는 경우: "벌점 이력이 없습니다." 표시
- 오더 ID는 오더 관리 ID 규칙(`O-9XXXX`)을 따름
- 오더 ID 클릭 시 새 탭으로 오더 관리 상세 화면으로 이동 (`?page=orders&orderId=...`)

### 3.3 벌점 값 분포 (목업)
- 전체 수행원의 50%는 벌점 0
- 나머지 50%에서 1~4 범위의 값 (정규분포 기반)
