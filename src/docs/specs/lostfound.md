# 분실물 관리 (Lost & Found)

분실물 관리는 세차 과정에서 발견된 고객의 분실물을 등록, 추적, 처리하는 기능입니다. 옥스트라(Oxtra) 시스템의 '분실물 카드'와 연동하여 고객에게 분실물을 안전하게 인계하는 것을 목표로 합니다.

---

## 1. 데이터 모델 (Data Model)

분실물 정보를 구성하는 데이터 필드는 다음과 같습니다.

| 필드명 (영문) | 필드명 (국문) | 데이터 타입 | 설명 | 수정 가능 |
| :--- | :--- | :--- | :--- | :--- |
| `id` | 분실물 ID | Number | 시스템에서 자동 생성되는 고유 식별자 | 불가 |
| `createdAt` | 접수 일시 | Datetime | 분실물이 시스템에 처음 등록된 시간 | 불가 |
| `itemCategory` | 분실물 구분 | String | `'일반'` 또는 `'귀중품'` | **가능** |
| `status` | 처리 상태 | String | 아래 [처리 상태](#2-처리-상태-정의) 참조 | 불가 (자동 전이) |
| `itemDetails` | 습득물 상세 정보 | String | 습득한 분실물의 구체적인 정보 (예: 검은색 아이폰 13) | **가능** |
| `itemPhotos` | 습득물 사진 | Array | 분실물 사진 URL 목록 | **가능** |
| `carId` | 차량 ID | Number | 분실물이 발견된 차량의 고유 ID | 불가 |
| `carNumber` | 차량 번호 | String | 분실물이 발견된 차량의 번호 | 불가 |
| `zoneId` | 존 ID | Number | 차량이 소속된 존의 고유 ID | 불가 |
| `zoneName` | 존 이름 | String | 존 명칭 (예: 성수 비즈밸리) | 불가 |
| `region1` | 지역1 | String | 존의 상위 지역 (예: 서울) | 불가 |
| `region2` | 지역2 | String | 존의 하위 지역 (예: 성동구) | 불가 |
| `partnerId` | 파트너 ID | Number | 세차를 수행한 파트너의 고유 ID | 불가 |
| `partnerName` | 파트너 이름 | String | 파트너 명칭 | 불가 |
| `relatedOrderId` | 연계 오더 ID | Number | 분실물이 발견된 세차 오더의 ID | 불가 |
| `lostItemCardReceiptNumber` | 분실물 카드 접수 번호 | String | 옥스트라 시스템과 연동되는 접수 번호 | 불가 |
| `deliveryAddress1` | 배송 주소1 | String | 배송지 기본 주소 (도로명 또는 지번) | **가능** |
| `deliveryAddress2` | 배송 주소2 | String | 배송지 상세 주소 | **가능** |
| `recipientName` | 수령인 이름 | String | 분실물 수령인의 이름 | **가능** |
| `recipientPhone` | 수령인 휴대폰 번호 | String | 분실물 수령인의 연락처 | **가능** |

### 1.1 필수 데이터 제약

- **연계 오더 ID (`relatedOrderId`)**: 모든 분실물은 세차 오더와 연계되어야 하며, 유효한 오더 ID를 가져야 합니다.
- **분실물 카드 접수 번호 (`lostItemCardReceiptNumber`)**: 모든 분실물은 옥스트라 시스템의 분실물 카드와 연동되어야 하며, 유효한 접수 번호를 가져야 합니다.

## 2. 처리 상태 정의 (Status Definition)

### 2.1 상태 목록

| 상태명 | UI Badge | 설명 |
| :--- | :--- | :--- |
| **배송지 미입력** | `warn` (황색) | 분실물은 등록되었으나, 고객의 배송지 정보가 아직 입력되지 않은 상태입니다. |
| **발송 대기** | `info` (청색) | 고객의 배송지가 입력되어 발송을 기다리는 상태입니다. (일반만 해당) |
| **발송 완료** | `ok` (녹색) | 고객에게 분실물 배송이 완료된 상태입니다. (일반만 해당) |
| **경찰서 인계** | `default` (회색) | 분실물을 관할 경찰서에 인계한 상태입니다. (귀중품만 해당) |
| **폐기 완료** | `default` (회색) | 보관 기간이 만료되어 규정에 따라 분실물을 폐기한 상태입니다. |

### 2.2 상태 전이 방식

처리 상태는 사용자가 직접 변경하지 않으며, 아래 조건에 따라 **자동 전이** 또는 **액션 버튼**으로 변경됩니다.

**자동 전이 (주소 저장 시):**
- **일반**: 배송 주소(주소1) 저장 시 `배송지 미입력` → `발송 대기` 자동 전이
- **귀중품**: 경찰서 주소(주소1) 저장 시 `배송지 미입력` → `경찰서 인계` 자동 전이

**액션 버튼:**
- **[발송 완료]**: `일반` + `발송 대기` 상태에서만 활성화, 클릭 시 `발송 완료`로 전환
- **[폐기 완료]**: 비종결 상태에서 활성화 (구분 무관), 클릭 시 `폐기 완료`로 전환

### 2.3 구분별 상태 흐름

**일반:**
- `배송지 미입력` →(주소 저장)→ `발송 대기` →([발송 완료] 버튼)→ `발송 완료`
- 어떤 비종결 상태에서든 `[폐기 완료]` 버튼으로 → `폐기 완료`

**귀중품:**
- `배송지 미입력` →(경찰서 주소 저장)→ `경찰서 인계`
- `배송지 미입력` 상태에서 `[폐기 완료]` 버튼으로 → `폐기 완료`

### 2.4 종결 상태

`발송 완료`, `경찰서 인계`, `폐기 완료`는 종결 상태입니다.
- 종결 상태에서는 분실물 구분 Select가 비활성화되며, 처리 상태는 Badge로만 표시됩니다.
- 배송 주소, 수령인 정보도 읽기 전용으로 표시됩니다.
- Drawer Footer의 액션 버튼([발송 완료], [폐기 완료])이 숨겨집니다.

### 2.5 구분 변경 시 상태 리셋

- 구분 변경 시 현재 상태가 새 구분의 유효 상태 목록에 없으면 `'배송지 미입력'`으로 자동 리셋됩니다.
- 컨펌 메시지에 상태 초기화 안내가 포함됩니다.

## 3. 미션 연동 로직 (Mission Integration)

- 세차 수행 전, 해당 차량(`car_id`)에 고객이 신고한 '분실물 카드'가 옥스트라 시스템에 존재하는 경우, 해당 세차 오더에 **'분실물 수거' 미션이 자동으로 등록**됩니다.
- 핸들러는 이 미션을 확인하고, 세차 시 분실물 수거에 더욱 주의를 기울여야 합니다.

---

## 4. 분실물 정보 생성 흐름 (Data Creation Flow)

분실물 정보는 아래 두 가지 경로를 통해 생성됩니다. 두 경우 모두 연계 오더 ID와 분실물 카드 접수 번호가 자동으로 설정되므로, 어드민에서 직접 수정할 수 없습니다.

### 4.1 옥스트라 분실물 카드가 존재하는 경우

1. 고객이 옥스트라(Oxtra) 시스템에 분실물 카드를 등록합니다.
2. 해당 차량(`carId`)에 세차 오더가 발행되면, 오더에 **'분실물 수거' 미션이 자동 등록**됩니다.
3. 수행원이 오더 수행 시, 분실물 카드 정보를 확인하고 차량에서 해당 분실물을 수거합니다.
4. 수행원 에이전트에서 분실물 수거를 완료하면, **차량관리 플랫폼에 분실물 정보가 자동 생성**됩니다.
   - `relatedOrderId`: 해당 세차 오더 ID
   - `lostItemCardReceiptNumber`: 옥스트라 분실물 카드 접수 번호

### 4.2 옥스트라 분실물 카드가 없는 경우 (수행원 현장 발견)

1. 수행원이 세차 과정에서 고객이 신고하지 않은 분실물을 발견합니다.
2. 수행원 에이전트에서 분실물 정보를 등록하면, **차량관리 플랫폼에 분실물 정보가 생성**됩니다.
3. 동시에 해당 오더 기준으로 **옥스트라 분실물 카드 정보가 자동 생성**됩니다. (데이터 동기화)
   - `relatedOrderId`: 해당 세차 오더 ID
   - `lostItemCardReceiptNumber`: 동기화 생성된 옥스트라 접수 번호

## 5. 기능 명세 (Feature Specifications)

### 5.1. 리스트 (DataTable)

- **조회 조건 설정 (FilterPanel)**
  - 공용 `FilterPanel` 컴포넌트 사용 (12-col 그리드, 칩 표시, 설정 초기화)
  - 필터 필드:
    1. **검색 항목** (Select): `차량 번호` / `분실물 카드` 선택
    2. **검색어** (Input): 선택한 검색 항목 기준 텍스트 검색
    3. **파트너 이름** (Select): 기본값 `전체`
    4. **처리 상태** (Select): 기본값 `전체`, 옵션: 배송지 미입력, 발송 대기, 발송 완료, 경찰서 인계, 폐기 완료
    5. **접수일 시작** (Date): 기본값 현재 기준 2개월 전
    6. **접수일 종료** (Date): 기본값 오늘

- **컬럼 (Columns)**
  1. **분실물 ID**: `id`
  2. **파트너 이름**: `partnerName`
  3. **차량 번호**: `carNumber`
  4. **존 이름**: `zoneName`
  5. **오더 ID**: `relatedOrderId` (클릭 시 해당 세차 오더 상세 페이지로 이동)
  6. **접수 일시**: `createdAt`
  7. **분실물 카드 번호**: `lostItemCardReceiptNumber` (값이 없으면 빈 칸으로 표시)
  8. **처리 상태**: `status` (Badge 적용)

- **정렬**: 컬럼 헤더 클릭 시 오름차순/내림차순 정렬 (기본: ID 내림차순)
- **페이지네이션**: 40건 단위 페이지네이션

### 5.2. 상세 조회 (Drawer)

- **Drawer 제목**: `"분실물 상세 정보"` / 부제: `"분실물 상세 및 처리 상태 관리"`
- **Footer**: 좌측에 액션 버튼 (`[발송 완료]`, `[폐기 완료]`), 우측에 `[닫기]` 버튼
  - `[발송 완료]` (primary): `일반` + `발송 대기` 상태에서만 표시
  - `[폐기 완료]` (danger): 비종결 상태에서 표시 (구분 무관)
  - 각 버튼 클릭 시 `confirm()` 확인 후 상태 변경
- **인라인 편집**: 수정 가능한 필드는 처음부터 Select / Input / Textarea로 표시되며, 별도의 수정 모드 진입 없이 즉시 편집 가능합니다.

- **카드 섹션 구성**
  1.  **분실물 정보**
      - `분실물 ID` (수정 불가, 텍스트)
      - `접수 일시` (수정 불가, 텍스트)
      - `분실물 구분`: `Select` — 값 변경 시 컨펌 후 즉시 반영
      - `처리 상태`: Badge 표시 (읽기 전용, 자동 전이 및 액션 버튼으로만 변경)
      - `배송 주소` (일반) / `경찰서 주소` (귀중품): 우편번호 검색(기본) / 직접 입력 라디오 선택, 주소1 + 주소2 입력 후 [저장] 버튼
        - `배송지 미입력` 상태에서 힌트 메시지 표시: "주소를 입력하고 저장하면 '발송 대기' 상태로 변경됩니다." (일반) / "경찰서 주소를 입력하고 저장하면 '경찰서 인계' 상태로 변경됩니다." (귀중품)
      - `수령인 이름`: `Input` + 개별 [저장] 버튼 (구분 무관)
      - `수령인 휴대폰 번호`: `Input` + 개별 [저장] 버튼 (구분 무관)
      - `습득물 상세 정보`: `Textarea` + 개별 [저장] 버튼
      - `습득물 사진`: 사진 추가/삭제 즉시 반영

  2.  **차량 정보**
      - `차량 번호` (수정 불가)
      - `존 이름` (수정 불가)
      - `지역1` (수정 불가)
      - `지역2` (수정 불가)
      - `존 ID` (수정 불가)

  3.  **세차 오더 정보**
      - `파트너 이름` (수정 불가)
      - `연계 오더 ID` (수정 불가)

  4.  **분실물 카드 정보 (옥스트라 연동)**
      - `카드 접수 번호` (수정 불가)

### 5.3. 인라인 편집 상세

#### Select 필드 (분실물 구분)
- 항상 Select 컴포넌트로 표시 (종결 상태 제외)
- 값 변경 시 `confirm()` 다이얼로그로 확인 → 확인 시 즉시 items에 반영

#### 처리 상태
- 사용자가 직접 변경 불가, 항상 Badge로 표시 (읽기 전용)
- 주소 저장 시 자동 전이, Drawer Footer 액션 버튼으로만 변경

#### Input/Textarea 필드 (상세 정보, 수령인 이름, 수령인 휴대폰 번호)
- 항상 입력 필드로 표시
- 각 필드 옆에 개별 [저장] 버튼 제공
- 변경사항이 없으면 [저장] 버튼 비활성화 (disabled)
- 변경사항이 있으면 [저장] 버튼이 primary 색상으로 활성화

#### 배송 주소 / 경찰서 주소
- 분실물 정보 카드 내 처리 상태 하단에 위치
- 귀중품일 경우 레이블이 `경찰서 주소`로 변경 (DB 필드는 동일)
- 입력 모드 선택 (라디오): `우편번호 검색` (기본) / `직접 입력`
- **우편번호 검색 모드**: 주소1은 readOnly (Daum Postcode API 결과 자동 입력) + [주소 검색] 버튼, 주소2는 수동 입력
- **직접 입력 모드**: 주소1, 주소2 모두 수동 입력, [주소 검색] 버튼 숨김
- [저장] 버튼 클릭 시 주소1/주소2 모두 저장
- `배송지 미입력` 상태에서 주소1 저장 시:
  - `일반` → `발송 대기`로 자동 전환
  - `귀중품` → `경찰서 인계`로 자동 전환

#### 수령인 정보
- 배송 주소 하단에 위치
- `수령인 이름`: Input + 개별 [저장] 버튼
- `수령인 휴대폰 번호`: Input + 개별 [저장] 버튼
- 분실물 구분(일반/귀중품) 무관하게 입력 가능
